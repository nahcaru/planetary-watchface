## Commands

```bash
# Develop (live preview via Zeus CLI)
npm run dev        # zeus dev

# Build for release
npm run build      # zeus build

# Regenerate planet assets (downloads from NASA/Wikimedia, crops & resizes to 480×480)
npm run download-assets   # uv run scripts/download_planets.py
```

The project uses `@zeppos/zeus-cli` (installed via `bun install` or `npm install`). The Zeus CLI handles sideloading to the device during `dev` and packaging for `build`.

## Architecture

This is a **ZeppOS 2.x watch face** for the Amazfit Balance (480×480 display).

- **`app.json`** — manifest: declares `appId`, targets (`480x480-amazfit-balance`, device sources for Monaco/MonacoWN/MonacoW), and runtime API version `2.0.0`.
- **`app.js`** — minimal app lifecycle shell required by ZeppOS; business logic lives entirely in the watch face module.
- **`watchface/index.js`** — the only watch face screen. Runs entirely in `onInit`/`onResume`/`onPause` lifecycle hooks:
  - Maps ZeppOS day-of-week (1=Monday … 7=Sunday) to a planet via `PLANET_BY_DAY`.
  - Renders: full-screen planet `IMG` → semi-transparent `FILL_RECT` scrim at bottom → planet name `TEXT` at top → large time `TEXT` inside scrim → date `TEXT` below time.
  - Time updates every 60 s via `setInterval` started in `onResume`, cleared in `onPause`.
- **`assets/480x480-amazfit-balance/`** — pre-rendered 480×480 PNGs (one per planet/celestial body). Regenerated by `scripts/download_planets.py`, which resolves Wikimedia Commons filenames via the Wikipedia API, downloads, center-crops, and resizes with Pillow.

## ZeppOS API notes

- Import UI primitives from `@zos/ui`, sensor data from `@zos/sensor`, timers from `@zos/timer`.
- Widget coordinates are absolute pixels; the canvas is exactly 480×480.
- `prop.TEXT` is used for runtime widget text updates (`widget.setProperty`).
- ZeppOS week convention: `1`=Monday, `7`=Sunday (not 0-indexed JS `Date.getDay()`).
